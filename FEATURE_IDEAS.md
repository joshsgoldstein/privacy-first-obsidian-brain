## Phase 4: UX Enhancements (High Priority)

### 1. Toggle RAG On/Off Per Message 🔥
**What:** Button/toggle in chat input to disable note retrieval for specific questions

**Use Case:**
- Ask LLM directly without retrieving notes
- General knowledge questions (not specific to your vault)
- Quick calculations or code generation
- When you want pure LLM without RAG

**UI Design:**
```
┌─────────────────────────────────────────────┐
│ Ask a question...                           │
│                                             │
│  [💾] [🔍] [📝]                    [Send ➤] │
└─────────────────────────────────────────────┘
   ^     ^     ^
   |     |     +-- Save as note
   |     +-------- Toggle RAG (highlighted = on)
   +------------- Attach file
```

**States:**
- 🔍 **ON** (default): Retrieves notes, shows sources
- 🔍 **OFF**: Sends directly to LLM, no retrieval
- Could show different indicator: "Searching your notes..." vs "Asking LLM..."

**Settings:**
- Default state (on/off)
- Remember last state per session
- Keyboard shortcut to toggle (Cmd/Ctrl+R)

**Priority:** HIGH - Simple to implement, huge UX win

### 2. Save Response as Note 🔥
**What:** One-click button to save assistant's response as a new markdown file

**Use Case:**
- Assistant writes something you want to keep
- Create note from generated content
- Build knowledge base from conversations
- Archive important answers

**UI Design:**
```
┌─────────────────────────────────────────────┐
│ Assistant                                   │
│                                             │
│ The Document Assembly Agent is...          │
│                                             │
│ Sources: [[note1]] [[note2]]               │
│                                             │
│ [💾 Save as Note] [📋 Copy] [🔄 Regenerate]│
└─────────────────────────────────────────────┘
```

**Behavior:**
- Click "Save as Note" button
- Opens modal:
  - Filename input (auto-generated from first line or timestamp)
  - Folder selection (default to chat folder or root)
  - Option: Include sources as links?
  - Option: Include original question as heading?
- Creates markdown file
- Opens file in editor (optional)
- Shows confirmation toast

**Generated Note Format:**
```markdown
# What is the Document Assembly Agent?

> Question asked: 2026-01-08 14:30

The Document Assembly Agent is...

## Sources
- [[Raw future state notes]]
- [[Taxonomy]]

---
*Generated by Smart Second Brain on 2026-01-08*
```

**Settings:**
- Default save location
- Auto-open after save (yes/no)
- Include metadata (yes/no)
- Filename template

**Priority:** HIGH - Frequently requested, connects chat to vault

## Technical Notes

### Save as Note Implementation
```typescript
async saveResponseAsNote(message: Message): Promise<void> {
  // Generate filename
  const filename = await this.promptForFilename(message);

  // Format content
  const content = this.formatAsNote(message);

  // Create file
  const file = await this.app.vault.create(
    `${this.settings.chatFolder}/${filename}.md`,
    content
  );

  // Optionally open
  if (this.settings.autoOpenSavedNotes) {
    await this.app.workspace.getLeaf().openFile(file);
  }

  new Notice(`Note saved: ${filename}`);
}
```

### Toggle RAG Implementation
```typescript
interface ChatInputState {
  ragEnabled: boolean;
}

async handleSend(text: string, state: ChatInputState): Promise<void> {
  if (state.ragEnabled) {
    // Normal RAG flow: retrieve → generate
    const results = await this.vectorStore.search(text);
    const context = this.formatContext(results);
    const response = await this.provider.generate(text, context);
    // ... show with sources
  } else {
    // Direct LLM: no retrieval
    const response = await this.provider.generate(text, '');
    // ... show without sources
  }
}
```

## User Stories

**As a user, I want to...**
1. Ask general questions without searching my notes
2. Save great responses as permanent notes in my vault
3. Quickly copy responses to use elsewhere
4. See what notes were used without opening them
5. Regenerate responses if I don't like the first one

**So that I can...**
- Use the LLM for both my notes AND general knowledge
- Build my knowledge base from conversations
- Integrate AI responses into my writing workflow
- Trust the sources without context switching
- Iterate on answers until they're perfect
